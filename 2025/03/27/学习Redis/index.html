一.Redis快速入门

1.初识Redis

Redis是键值数据库—》NoSql

1.1 认识NoSQL

1.2 认识Redis

Redis远程词典，是一个基于内存的键值型NoSQL数据库。

特征：

​ 1.键值型（key-value）型，value支持多种不同数据，功能丰富。

​ 2.单线程，每个命令具备原子性。

​ 3.低言辞，速度快（基于内存，内存存取速度快）

​ 4.支持数据持久化，多语言客户端。

1.3 安装Redis

Windows下安装和配置Redis_redis下载安装配置windows-CSDN博客

2.Redis的常见命令

2.1 临时服务

启动临时服务：redis-cli

检测服务连通性：ping 返回PONG则说明连接成功

关闭临时服务：shutdown

2.2 永久服务

启动永久服务：redis-server.exe –service-install redis.windows.conf –loglevel verbose

启动服务：redis-server –service-start

停止服务：redis-server –service-stop

卸载服务：redis-server –service-uninstall

3.Redis数据结构介绍

Redis是一个key-value的数据库，key一般是String类型，不过value的类型多种多样。

3.1 通用命令

help [command]可以查看一个指令的具体用法。

1.help keys 查看符合模板的所有key,不建议在生产环境设备上使用—->keys pattern

2.help del 删除指定一个或多个key—->del key

3.help exists 判断key是否存在—–>exists key存在返回1，不存在返回0

4.help expire 给一个key设置有效期，有效期到期时该key会被自动删除。（节省内存空间）—>expire key seconds

5.help ttl 查看一个key的剩余有效期。—>ttl key

3.2 String类型的常见命令

String类型的三种格式：字符串，int,float

3.4 Key的层级格式

Redis的key允许有多个单词形成层级结构，多个单词之间用’:’隔开。

eg.项目名：业务名：类型：id

3.5 Hash类型的常见命令

Hash类型，也叫散列，其value是一个无序字典，类似于Java中的HashMap结构。

3.6 List类型的常见命令

Redis中的List类型与java中的LinkedList类似，可以看作是一个双向链表结构，既可以支持正向检索也可以支持反向检索。

特征：有序，元素可以重复，插入和删除快，查询速度一般。

通常用来保存一些有序数据。eg.朋友圈点赞列表，评论列表等。

3.7 Set类型的常见命令

Redis中的set结构与java中的HashSet类似，可以看作是一个value为null的HashMap。因为也是一个hash表，因此具备了与HashSet类似的特征。

特征：无序，元素不可重复，查找快，支持交集，并级，差集等功能。

3.Redis进行缓存功能

3.1 什么是缓存

缓存就是数据交换的缓冲区（称作Cache）,是存储数据的临时地方，一般读写性能高。

缓存的作用：1.降低后端负载，2.提高读写效率，降低响应时间。

缓存的成本：1.数据一致性成本，2.代码维护成本，3.运维成本。

3.4 缓存穿透（面试常问）

缓存穿透是指客户端请求的数据在缓存中和数据库中都不存在，不断发起这样的请求，给数据库带来巨大的压力。

不怀好意的人，一直拿多个线程一直访问缓存和数据库中不存在的数据（id），这样就会导致每一次请求，都会让数据库处理，导致数据库崩溃。

![Redis架构图](/images/image-20250304165725323.png” alt=”image-20250304165725323” )

1.缓存空对象：优点：实现简单，维护方便。

​ 缺点：额外的内存消耗，可能造成短期的不一致。

5.Feed流

关注推送也叫做Feed流，直译为投喂。为用户持续的提供“沉锓式”的体验，通过无限下拉刷新获取新的信息。

5.1 Feed流的俩种常见模式：

（1）Timeline

timeline 不做内容筛选，简单的按照内容发布时间排序，常用于好友或关注。例如朋友圈。

优点：信息全面，不会有缺失。并且实现也相对简单。

缺点：信息噪音较多，用户不一定感兴趣，内容获取效率低。

timeline模式的实现方案三种：

拉模式（也叫做读扩散）

推模式（也叫写扩散）

推拉结合

（2）智能排序

利用智能算法屏蔽掉违规的，用户不感兴趣的内容。推送用户感兴趣信息来吸引用户。

优点：投喂用户感兴趣信息，用户黏度很高，容易沉迷

缺点：如果算法不精准，可能起到反作用。

（3）Feed流的分页问题

Feed流中的数据会不断更新，所以数据的角标也在变化，因此不能采用传统的分页模式（按索引分页）。

Feed流的滚动分页

滚动分页查询参数：

max：当前时间戳 | 上一次查询的最小时间戳

min：0

offset：偏移量 0 | 在上一次的结果中，与最小值一样的元素个数

count：查询条数
