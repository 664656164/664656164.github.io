数据库基础知识

一.mysql数据库的使用

1.mysql服务的启动和关闭：

a.启动mysql服务

net start mysql80

b.关闭mysql服务

net stop mysql80

2.登录mysql数据服务器：

mysql -u root -p;

再输入密码；

3.注意：

mysql> 指提示符

mysql命令语句以分号“;”结束，按回车键可执行；

mysql命令不区分大小写;

在命令行客户端输入的语句可以通过上，下方向键进行查找；

4.字符集：

5.操作：

查看mysql的状态信息：mysql> status;

查看数据库： show databases；

6.数据类型：

1.datetime不赋值，自动返回null；

timestamp不赋值，自动返回系统实时时间

2.char是定长，长度确定，如果没有达到它的长度，它会自动补空格，用于储存小于等于该长度的字符串

varchar是变长，根据你输入字符串的大小，它自己给你划分长度。

3.int和int(2)的区别

也就是说int(2)只能存放小于100的数，如果只有一位数字，那么主动加前导0，比如2-》02；int(2)代表宽度为2.

int(M)里面这个M和存储大小无关。M写成20它也是4个字节的。

6.设置默认值

a.在创建表时，设置默认值：

age int default 25(设置年龄默认值为25)

b.如果已经有表了，想设置默认值：(modify或者change或者alter column)

​ alter table my_table;

​ alter column age set default 25;

二.mysql数据库的基本操作：

1.创建数据库：

create database 数据库名称；

2.查看数据库：

show databases；

3.选择数据库：

use 数据库名称;(database changed表示选择数据库成功)

4.删除数据库：

drop database 数据库名称;

1.创建数据库表：

数据表命名原则：长度最好不超过30个字符；多个单词之间使用下划线分割，不允许有空格；不允许将mysql作为关键字; 不允许与同一数据库中其它数据表同名。

desc 表名；（describe 表名;）

5.修改字段数据类型：

8.修改表名：

alter table 旧表名 rename [to] 新表名;

9.删除表：

drop table [if exists] 表名;

10.删除指定表并重新创建表

truncate table 表名；

在删除表时，表中的全部数据也会被删除;

11.查看数据库中的所有表

*.表的数据操作:

1.添加数据

a.给指定字段赋值：insert into 表名（字段1，字段2，……） values(值1，值2…..)

b.给全部字段添加数据：insert into 表名 values(值1，值2….) 注意与表得字段一一对应

c.批量添加数据：insert into 表名（字段1，字段2，……） values(值1，值2…..)，values(值1，值2…..)……;

​ insert into 表名 values(值1，值2….) ,values(值1，值2…..)……;

2.修改数据

格式： update 表名 set 列名1 = 新值1, 列名2 = 新值2, … WHERE 条件;

update students set age = age + 10, name = ‘John Doe’ where id = 1;

where子句指定了要更新的记录的条件，即id等于1。这将更新id为1的学生的年龄增加10岁，并将名字设置为”John Doe”。

3.删除数据

格式：delete from 表名 where 条件

delete from student where age>18; （这将删除年龄大于18岁的学生记录。）

一.单表查询(select)

1.查询多个字段

select *from 表名;//查询所有列

2.设置别名

select 字段1（as 别名1)，字段2 （as 别名2）…….from 表名;

3.去除重复记录

4.条件语句

select 字段列表 from 表名 where 条件列表

注意：数据库中的等于是单等号。

in(x,y,z)==x||y||z;

2.注意：

三.分组查询(group by)

1.用法：

select 字段列表 from 表名 【where 条件】group by 分组字段名 【having 分组后过滤条件】;

2.where和having得区别：

执行时机不同：where是分组之前进行过滤，不满足where条件，不参与分组；而having是分组之后对结果进行过滤。

判断条件不同：where不能对聚合函数进行判断，而having可以。

3.注意：

执行顺序：where>聚合函数>having

分组之后，查询的字段一般为聚合函数和分组字段，查询其它字段无任何意义。

*四.排序查询（order by）

1.用法：

select 字段列表 from 表名 order by 字段1 排序方式1，字段2 排序方式2；

2.排序方式

asc:升序（order by 默认值）

desc:降序

3.注意

如果是多字段排序，当第一个字段值相同时，才会根据第二个字段进行排序。
